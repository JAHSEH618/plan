# 事业部 AI 需求分类表

类别口径：

- 需要工程实现：需要系统、API、数据库、RPA、权限、部署或前后端工程交付。
- skill沉淀实现：适合沉淀为可复用 AI 工作流、Codex/Claude Code skill、提示词模板、脚本包或检查清单。
- 部门内部人工规范问题：核心矛盾是标准、口径、职责、模板或数据维护机制不清，先做内部规范，不优先工程化。
